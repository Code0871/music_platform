# Технический стек проекта

## 1. Языки программирования
| Язык | Роль |
|---|---|
| **Go** | Основной язык API и микросервисов (DDD) |
| **Rust** | Критически важные по производительности участки (стрим-процессинг, инференс легких моделей) |
| **Python** | ETL, ML-эксперименты, обучение моделей, Airflow DAGs |

## 2. Базы данных
| База | Назначение |
|---|---|
| **PostgreSQL** | Основной источник истины (OLTP), DB-per-service |
| **Milvus** | Векторная БД для поиска эмбеддингов треков/пользователей (грубый кандидат-поиск) |
| **ClickHouse** | OLAP для ML-обучения |
| **StarRocks** | Боевая real-time аналитика |
| **Redis** | Кеширование лент пользователей, онлайн-хранилище Feature Store, брокер результатов |

## 3. Хранилище данных и Lakehouse
| Компонент | Назначение |
|---|---|
| **MinIO** | Основное объектное хранилище (Data Lake) |
| **Apache Iceberg** | Табличный формат для MinIO (ACID, time-travel, единый каталог для StarRocks/ClickHouse/Spark) |
| **Apache Spark** | Распределенная обработка данных в батчах (через Airflow) |

## 4. Стриминг и CDC
| Компонент | Назначение |
|---|---|
| **RedPanda** | Основная шина данных (стриминг событий) |
| **Debezium** | CDC для захвата изменений PostgreSQL и публикации в RedPanda |
| **RisingWave / Arroyo (Rust)** | Стрим-процессинг (real-time агрегации фич, джойны потоков до попадания в StarRocks/Feast) |

## 5. ML-платформа
| Компонент | Назначение |
|---|---|
| **Feast** | Feature Store (офлайн- и онлайн-сервинг фич для обучения и инференса) |
| **MLflow** | Трекинг ML-экспериментов и реестр моделей |
| **BentoML / Triton Inference Server** | Сервинг ML-моделей в продакшене (gRPC/REST) |
| **Airflow** | Оркестрация ETL/ELT, обучения моделей, обновления фич в Feast |

## 6. Инфраструктура и сетевое взаимодействие
| Компонент | Назначение |
|---|---|
| **Nginx** | Граничный Ingress Controller (SSL, статический роутинг) |
| **Kong / KrakenD / APISIX** | API Gateway (аутентификация, canary-релизы, роутинг в Service Mesh) |
| **Consul** | Service Discovery и динамическое управление конфигурациями |
| **Istio / Linkerd** | Service Mesh (mTLS, наблюдаемость, трафик-менеджмент между Go-сервисами) |

## 7. Наблюдаемость
| Компонент | Назначение |
|---|---|
| **OpenTelemetry** | Сбор трейсов и метрик из Go/Rust/Python-сервисов |
| **Prometheus + Grafana** | Метрики инфраструктуры и бизнес-метрик (CTR, latency) |
| **Grafana Loki** | Агрегация и поиск логов |
